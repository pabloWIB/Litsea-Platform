# 10 · Internacionalización (i18n)
**Librería:** next-intl  
**Idiomas:** Español (default) · English · Français  
**Archivos:** `messages/es.json` · `messages/en.json` · `messages/fr.json`

---

## Configuración

**`localePrefix: 'as-needed'`** — el español no lleva prefijo en la URL:
```
/ → español (default)
/en/ → inglés
/fr/ → francés
```

**Detección automática de locale:** next-intl detecta el locale del navegador pero respeta el prefijo de URL.

## Cómo usar i18n en el código

### Server Component (RSC)
```typescript
import { getTranslations } from 'next-intl/server'

export default async function Page() {
  const t = await getTranslations('dashboardTerapeuta')
  return <h1>{t('bienvenido', { name: 'María' })}</h1>
}
```

### Metadata en Server Component
```typescript
export async function generateMetadata(): Promise<Metadata> {
  const t = await getTranslations('pageTitles')
  return {
    title: t('login'),
    description: t('loginDesc'),
  }
}
```

### Client Component
```typescript
'use client'
import { useTranslations } from 'next-intl'

export default function LoginForm() {
  const t = useTranslations('auth')
  return <button>{t('signIn')}</button>
}
```

### Variables en traducciones
```typescript
// En el JSON:
"bienvenido": "¡Bienvenido/a, {name}!"

// En el código:
t('bienvenido', { name: profile.full_name })
// → "¡Bienvenido/a, María!"
```

### Rich text (links, bold)
```typescript
// En el JSON:
"checkEmailMessage": "Enviamos un enlace a <b>{email}</b> para restablecer tu contraseña."

// En el código (Client Component):
t.rich('checkEmailMessage', {
  email: 'maria@email.com',
  b: (chunks) => <strong>{chunks}</strong>
})
```

## Convenciones

1. **Un namespace por dominio funcional** — no por componente ni por página
2. **Usar `{variable}` para interpolación** — nunca concatenar strings
3. **Rich text solo con `t.rich()`** — nunca `dangerouslySetInnerHTML`
4. **ES primero** — agregar a `es.json`, luego traducir a `en.json` y `fr.json`
5. **`pageTitles`** centraliza todos los `<title>` y `<meta description>`
6. **Plurales**: `"{count} aplicaciones"` — el componente decide si es 1 o más

---

## Cómo agregar un namespace nuevo

1. Agregar el namespace a `messages/es.json`
2. Copiar estructura idéntica a `messages/en.json` (traducir valores)
3. Copiar a `messages/fr.json` (traducir valores)
4. Usar en el componente con `getTranslations` o `useTranslations`
5. Actualizar este documento (`10-i18n.md`) con las nuevas claves
